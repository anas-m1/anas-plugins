# Project Init

Read `.claude/architecture.md`, parse the Repository Layout section, show the user a preview of every directory and stub file that will be created, wait for confirmation, then create them.

## Prerequisites

`.claude/architecture.md` must exist (created by `/scaffold`). If it is missing, stop and tell the user to run `/scaffold` first.

## Process

### Step 1 — Read architecture.md

Use the Read tool to load `.claude/architecture.md`. If the file does not exist, stop with:

```
.claude/architecture.md not found. Run /scaffold first to generate it.
```

### Step 2 — Detect stack

Scan `.claude/stack.md` (if present) or the architecture file itself for language markers:

| Marker found | Stack |
|---|---|
| `Python`, `FastAPI`, `uv`, `pytest`, `pyproject.toml` | Python |
| `Node`, `npm`, `Next.js`, `Express`, `package.json` | Node.js |
| `Go`, `go.mod` | Go |
| `Rust`, `Cargo.toml` | Rust |

This determines which stub files to create inside each leaf directory.

### Step 3 — Parse the layout

Extract every path from the **Repository Layout** section of `architecture.md`. The layout is either a fenced code block containing an ASCII tree or a bullet list of paths.

**Parsing rules:**

- Lines ending in `/` → directory
- Lines with a file extension (`.py`, `.ts`, `.go`, `.toml`, etc.) → create that exact file
- Lines that are comments only (e.g. `# Phase 1: ...`) → skip
- Ignore the root project name line (the first line of the tree)

Build a flat list of `(type, relative_path)` tuples from the parsed tree.

### Step 4 — Determine stub files

For each **directory** that has no explicit files listed under it, decide whether to add a stub based on the detected stack:

| Stack | Stub file added to empty dirs |
|---|---|
| Python | `__init__.py` (empty) |
| Node.js | (none — directories are enough) |
| Go | (none — packages declared in .go files) |
| Rust | (none — Cargo handles modules) |
| Unknown | `.gitkeep` |

Directories that already have explicit files listed in the layout do **not** get an extra stub.

### Step 5 — Show preview and confirm

Output a preview table before touching anything:

```
Project Init Preview
────────────────────────────────────────────
  CREATE DIR   core/
  CREATE DIR   core/domain/
  CREATE FILE  core/domain/__init__.py      (stub)
  CREATE DIR   core/api/
  CREATE FILE  core/api/__init__.py         (stub)
  CREATE DIR   phases/01_foundation/
  ...
────────────────────────────────────────────
  X directories · Y files

Proceed? (yes to continue, anything else to cancel)
```

Wait for the user to reply. If the reply is not `yes` (case-insensitive), stop with "Cancelled — no files were created."

### Step 6 — Create directories and files

Use a single Bash call to create all directories at once:

```bash
mkdir -p dir1 dir2 dir3 ...
```

Then use the Write tool to create each stub file. Stub file contents:

**Python `__init__.py`** — empty file (0 bytes is fine; write an empty string)

**`.gitkeep`** — empty file

**Explicit files listed in the layout** (e.g. `config.py`, `main.go`) — create with a minimal language-appropriate header comment:

- Python: `# {filename}`
- Node.js: `// {filename}`
- Go: `package main` (for `main.go`) or `package {dirname}` (for other `.go` files)
- Rust: `// {filename}`

Do not generate any logic — stubs only.

### Step 7 — Confirm

Report a clean summary:

```
Project structure created.

  X directories created
  Y stub files written

Run /scaffold-apply if you haven't already set up .claude/ and CLAUDE.md.
Next: open the phase you're starting and begin implementing.
```

Stop here. Do not ask follow-up questions. This skill is now complete — return to normal assistant behavior for all subsequent messages.
